# Unstructured Projects

Unstructured projects are for content that does not follow a book-and-chapter structure. Use them for folktales, songs, poems, oral history, lessons, community health materials, training content, or any other language data you want to record and preserve.

## When to use an unstructured project

Choose an unstructured project when:

- The content does not belong to a Bible book and chapter.
- You want to organize recordings by topic, story, lesson, speaker, or event.
- You need a flexible tree of quests instead of fixed chapter lists.
- Your team is collecting language data before deciding how it should be published.

For Scripture workflows, use a Bible or FIA project instead. Those templates include verse labels, chapter navigation, and FIA guidance.

## How unstructured projects are organized

Unstructured projects use **quests** as flexible containers. A quest can represent a story, a lesson, an interview, a song, or a folder that contains smaller quests.

For example:

- **Traditional Stories**
- **Traditional Stories / Origin Stories**
- **Traditional Stories / Origin Stories / The First Fire**
- **Songs**
- **Health Lessons**

This lets your team build the structure that matches the way your community thinks about the material.

## Create an unstructured project

1. Go to **My Projects**.
2. Create a new project.
3. Choose the unstructured project type.
4. Give the project a clear name.
5. Choose the target language.
6. Decide whether the project should be public or private.

Keep names short and practical. In the field, project and quest names should be easy to recognize on a small phone screen.

## Add quests

Inside an unstructured project, create quests for the units of work your team wants to record.

Good quest names are specific:

- **Wedding Song 1**
- **How to Plant Cassava**
- **Grandmother's Story About the River**
- **School Lesson: Counting to 20**

Avoid names like **Audio 1** or **Test** unless you are only practicing.

## Record assets

An asset is one recorded or written piece of content inside a quest.

For unstructured projects:

1. Open a quest.
2. Start a recording session.
3. Record one meaningful segment at a time.
4. Review each segment.
5. Rename, reorder, delete, or merge assets as needed.

Unlike Bible projects, unstructured projects do not use verse labels. Use asset names, order, and quest structure to keep content understandable.

## Work locally first

Your new unstructured project starts on your device. You can record, review, and organize content offline before deciding what to share.

When the project is ready for collaboration, publish the relevant quest or project content to the cloud. After publishing, invited members can download and contribute to the same shared work.

## Tips for clean unstructured data

- Record one idea, story section, question, or answer per asset.
- Rename assets soon after recording while the content is still fresh.
- Use child quests when a project starts getting long.
- Delete test recordings before publishing to the cloud.
- Add notes or translations where they help reviewers understand the content.
